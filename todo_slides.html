Nesting Modularity

Not everything needs to be nested.
These are often your "base," or default, styles (ex. body, a, p).

Start to think in terms of reusable components.
Buttons, media objects, cards.

Use minimal specificity with selectors.
How might you combine multiple selectors to achieve an effect?

If an element's style is specific to the component,
it's a perfect candidate for nesting.

Nesting Modularity


                .btn {
                  // base button styles
                  // padding, border-radius, text-align, etc.
                }

                .btn-primary {
                  // color, background-color, border-color
                }

                .btn-large {
                  // font-size, line-height, base padding overrides
                }

                // <a class="btn btn-secondary" ...

This is a module.

No nesting required.

We're relying on the cascade.

Reusable, combinable components.
